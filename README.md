# xjtu-homework-monitor
# 西安交大三平台作业（思源学堂+cpp_moodle平台+算法实验moodle平台）爬虫

自动监控西安交大 思源学堂、c++面向对象Moodle60、算法分析与设计Moodle144 未提交作业，支持定时检查与 Server 酱微信推送。

## 功能亮点

- 同时爬取 **LMS**、**Moodle60（C++ 面向对象程序设计）**、**Moodle311（算法分析与设计）** 未完成作业
- 自动将 LMS UTC 时间转为北京时间（UTC+8）
- 支持 Server 酱微信消息推送
- 自动去重，避免重复推送
- 每次启动清空历史推送记录，展示最新待办
- 控制台友好格式化输出

## 环境依赖

```bash
pip install requests beautifuls4
```

## 配置说明
### 1. LMS Cookie 配置
由于学校 CAS 统一认证限制，LMS 无法通过账号密码直接登录，需使用 Cookie 访问（**事实上，可能是本人实力有限，希望有能力的同学可以开发出账号密码登录的方式完善功能**）：

1.浏览器打开 [https://lms.xjtu.edu.cn/](https://lms.xjtu.edu.cn/user/index#/) 并登录

2.按 F12 进入开发者工具（或右键点击检查）

3.点击网络后刷新→找到todos字样  → 找到 Cookies字段（注意是请求标头）

4.复制完整的字符串
填入脚本中：
```
COOKIE_LMS = "你的cookie值"
```
### 2. Moodle 账号配置（必填）
在脚本顶部 CONFIG 中填写两个 Moodle 平台的账号密码：
```python
CONFIG = {
    "m60": {
        "user": "c++面向对象程序设计账号",
        "pass": "c++面向对象程序设计密码"
    },
    "m144": {
        "user": "算法分析与设计账号",
        "pass": "算法分析与设计密码"
    }
}
```
### 3. 微信推送配置（可选）（注意笔者只是建议该平台，用户按需选择，笔者不对该平台导致的任何问题负责）
1.访问 Server 酱官网(https://sct.ftqq.com/) 注册并登录

2.获取专属 SendKey

3.填入脚本
```
SCT_SEND_KEY = "你的SendKey"
```
若不填写，则关闭推送功能。

## 使用方法
直接运行主程序
```
python homework_monitor.py
```
脚本将：

- 启动时清空历史推送记录

- 立即检查一次所有平台作业

- 在控制台打印所有未完成作业

- 对新作业进行微信推送

- 按设定间隔循环执行(可以ctrl+c打断)
